# Contributing

Thanks for helping out.

- Open an issue before a large change so we can align on direction.
- Keep pull requests focused: one purpose per PR.
- Match the existing style of the files you touch.
- Don't commit secrets, tokens, or credentials.
- Conventional commit messages are appreciated (`feat:`, `fix:`, `docs:`, `chore:`).

CI runs on every push and pull request; keep it green.
